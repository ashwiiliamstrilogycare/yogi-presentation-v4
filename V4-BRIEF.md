## Three v4 alignment shifts (highest priority)

### 1. Yogi = the brain · TC Workspaces = the body

> Yogi is the AI brain. **TC Workspaces is the body** — Mike Wise's platform that consumes Yogi's backend: memory, agents, MCP tools, evals. The staffer interacts with Workspaces. Yogi is the engine inside.

Anywhere a slide says "TC Assistant" — keep it, but make clear that Yogi is the underlying intelligence. Anywhere the platform is talked about, foreground the backend dependency:
- Memory layer (markdown, git, persistent)
- Agent layer (1,000 specialists, on-demand)

- Eval layer (12 categories, 95.4% mean)

### 2. NO "forest" vocabulary

Already scrubbed. Replacements used:
- "Forest Crew" → **"Specialist Crew"**
- "the forest" → **"the crew"** or **"the system"**
- "FOREST" → **"SPECIALIST"**
- File paths: `forest-crew/` → `specialists/`, `forest-memory/` → `memory/`

If you write new content, use **Specialist Crew · Crew · Sub-Agents · Specialists** vocabulary. Bear is still Yogi. Animals (Rocco/Otto/Bram etc.) are still named animals — just not "forest" anymore.

### 3. Productivity + Efficiency · **Companion-type interaction**

Yogi was designed to **multiply staffer output, not replace them**. The merger of Yogi inside Workspaces Home gives operators **supercharged ability via a companion-type interaction** — Yogi works *with* you, not *for* you. Same desk. Same screen. Same flow. Frame everything in terms of:
- Time saved (minutes-of-doing → seconds-of-checking)
- Decisions per hour
- Tasks completed per FTE-day
- Cycles cut
- Throughput uplift

Where possible, add a small "productivity receipt" to every applicable slide — a stat that ties Yogi to operator output.
